# 拆弹专家

大家好！这是我做的一个微信小程序小游戏。  
我今年 11 岁，这个游戏一开始叫“ABC猜数字”，后来我把它做成了一个“拆弹专家”的主题：玩家要在倒计时结束前猜出 4 位密码，猜对了就拆弹成功，猜错太多次就会爆炸。

## 我为什么做这个游戏

我想做一个既好玩又能锻炼思维的小游戏。  
这个游戏不是只靠运气，还要根据每次的反馈来推理，慢慢排除错误答案。

## 首页 Index 截取说明

小程序只有一个主要页面入口：

```text
pages/index/index
```

首页上主要有这些按钮：

| 首页按钮 | 作用 |
| --- | --- |
| 开始挑战 | 系统随机生成 4 位密码，玩家开始拆弹 |
| 拆弹训练 | 一共 100 关，从简单到困难练习推理 |
| 队友切磋 | 一个人设置密码，另一个人来猜 |
| 战绩排行 | 查看以前的挑战记录 |
| 拆弹手册 | 查看详细游戏规则 |
| 我的勋章 | 查看已经获得和还没获得的勋章 |
| 静音按钮 | 打开或关闭背景音乐 |

进入“开始挑战”以后，页面中间是炸弹控制面板。  
最大的数字区域就是输入 4 位密码的地方，右边的红色圆形按钮是“确认”。下面会显示每次尝试的密码和反馈。

## 游戏规则

系统会随机生成一个 4 位密码。密码可以 0 开头，比如 `0472` 是可以的。  
但是 4 个数字不能重复，比如 `1123` 不可以。

玩家每次也要输入一个不重复的 4 位数字。  
如果输入了重复数字，游戏会提示：

```text
数据不能重复，请重新输入
```

## A、B、C 是什么意思

每次猜完以后，系统会给出 A、B、C 的反馈：

| 反馈 | 意思 |
| --- | --- |
| A | 数字正确，位置也正确 |
| B | 数字正确，但是位置不正确 |
| C | 这个数字不在答案里 |

比如答案是 `1234`，我猜 `1325`：

| 数字 | 判断 |
| --- | --- |
| 1 | 数字对，位置也对，所以是 A |
| 3 | 答案里有，但是位置错了，所以是 B |
| 2 | 答案里有，但是位置错了，所以是 B |
| 5 | 答案里没有，所以是 C |

所以结果显示为：

```text
ABBC
```

注意：游戏不会写成 `1A2B1C`，而是直接显示成 `ABBC`。

## 怎么赢

如果猜的 4 位密码和答案完全一样，就会显示：

```text
AAAA
```

这就说明拆弹成功了！游戏会显示这次一共猜了多少次。

## 倒计时

每一局有 10 分钟倒计时。  
每猜错一次合法的密码，时间会减少 1 分钟。  
如果时间归零，任务失败，会出现爆炸效果。

## 拆弹训练

训练模式一共有 100 关。  
每一关会给一个线索，比如某个数字和答案的反馈是 `AABB`、`ABBC` 或 `CCCC`。玩家要根据线索推理出答案。

关卡难度会慢慢增加。已经闯过的关可以再挑战一次，而且训练模式不计入战绩排行。

## 勋章系统

游戏里还有“我的勋章”。  
完成一些目标可以获得勋章，比如连续挑战成功、完成 100 关训练、队友切磋连续成功等。

有些勋章可以获得多枚，数量会显示在勋章右下角。还有一个隐藏彩蛋勋章，获得之前不会告诉玩家。

## 项目文件

这个项目是微信小程序，主要文件在：

```text
app.json
pages/index/index.wxml
pages/index/index.wxss
pages/index/index.js
utils/game.js
utils/medals.js
```

其中：

| 文件 | 作用 |
| --- | --- |
| `pages/index/index.wxml` | 页面结构 |
| `pages/index/index.wxss` | 页面样式和动画 |
| `pages/index/index.js` | 游戏页面逻辑 |
| `utils/game.js` | 密码生成、判断 A/B/C、关卡线索 |
| `utils/medals.js` | 勋章获得规则 |
